# Meeting 页面 AI 集成说明

## 功能概述

在 Meeting 页面中集成了 AI 生成会议通知的功能，用户可以选择使用**代码生成**或 **AI 生成**两种方式。

## 使用方法

### 1. 填写会议信息

在 Meeting 页面填写以下信息：
- **会议链接1** 和 **会议号1**（必填）
- **会议主题A**（可选，默认：结营&答疑）
- **会议链接2** 和 **会议号2**（必填）
- **会议主题B**（可选，默认：训练营结营&项目成果展示）

### 2. 选择生成方式

页面底部有一个切换开关：
- **代码模式**（默认）：使用确定性算法生成，毫秒级响应
- **AI 模式**：使用 AI 智能生成，支持更灵活的输入理解

### 3. 生成通知

点击"立即生成"按钮，系统会根据选择的模式生成会议通知。

## 功能对比

| 特性 | 代码生成 | AI 生成 |
|------|---------|---------|
| 响应速度 | 毫秒级 | 秒级 |
| 确定性 | 完全确定 | 可能有变化 |
| 格式一致性 | 100% | 95%+ |
| 灵活性 | 固定规则 | 自然语言理解 |
| 成本 | 无 | API 调用费用 |
| 离线使用 | ✅ | ❌ |

## 技术实现

### AI Skill

创建了 `meeting-notice-generator` skill，包含完整的业务规则：
- 时间计算规则（周末/工作日/周五）
- 会议通知模板
- 参数解析逻辑

### 代码集成

1. **状态管理**：新增 `useMeetingAI` 状态控制生成方式
2. **AI 生成函数**：`generateMeetingNoticeWithAI` 调用 AI API
3. **UI 组件**：添加切换开关和生成按钮
4. **流式响应**：支持实时显示 AI 生成的内容

## 注意事项

1. **AI 模式依赖**：
   - 需要配置 Coze 或豆包 API
   - 需要网络连接
   - 可能产生 API 调用费用

2. **推荐使用场景**：
   - **代码模式**：日常快速生成，标准格式
   - **AI 模式**：测试对比，特殊需求

3. **性能优化**：
   - AI 生成采用流式响应，逐步显示结果
   - 支持 Coze 和豆包两种 AI 引擎
